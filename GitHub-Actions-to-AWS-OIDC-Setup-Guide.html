OIDC Setup Guide

Configure keyless authentication from GitHub Actions to AWS — no hardcoded secrets, no long-lived credentials. Just secure, temporary access via OpenID Connect.

1
Add identity provider in AWS IAM
AWS Console

Register GitHub's OIDC provider in your AWS account so AWS can verify tokens issued by GitHub Actions runners. This is a one-time setup per AWS account.

config
# In AWS Console → IAM → Identity providers
Provider URL: https://token.actions.githubusercontent.com
Audience: sts.amazonaws.com
📎
Full guide: docs.github.com → oidc-in-aws
2
Create IAM policy
IAM → Policies

Define the minimum permissions this deployment role needs — such as pushing images to ECR and updating ECS services. Follow least-privilege principles.

yaml
# Policy name
name: github-deployment-policy

# Attach permissions for ECR + ECS access
3
Create IAM role
IAM → Roles

Create a role with a trust policy scoped to your specific GitHub repository. Then attach the policy from step 2. The role ARN is what the workflow will assume.

yaml
name: github-deployment-role

# Trust policy condition (restrict to your repo)
token.actions.githubusercontent.com:sub:
  repo:your-org/your-repo:ref:refs/heads/main

# Resulting ARN
arn: arn:aws:iam::194343789279:role/github-deployment-role
4
Configure credentials in GitHub Actions
.github/workflows

Add this step before any AWS commands in your workflow. The action exchanges the GitHub OIDC token for temporary AWS credentials via STS — injected automatically into all subsequent steps.

yaml
- name: Configure AWS credentials (OIDC)
  uses: aws-actions/configure-aws-credentials@v4
  with:
    role-to-assume: arn:aws:iam::194343789279:role/github-deployment-role
    role-session-name: gha-ecr-ecs-deploy
    aws-region: ${{ env.AWS_REGION }}
💡
Temporary credentials are valid for 1 hour by default and automatically available to all subsequent steps in the job via environment variables.
